01 — System Architecture

The Full Stack Picture

TownHall's architecture follows a serverless-first, BaaS-heavy philosophy. This is the correct approach for a zero-cost launch — you write application logic, not infrastructure. The diagram below shows all layers and where each service lives.

Key Architectural Decision

All third-party API calls (Razorpay, Resend, Cloudinary signed uploads) MUST go through Supabase Edge Functions — never directly from the browser. This protects your API keys and enables server-side validation. Your current Cloudinary unsigned upload is the biggest security gap to close immediately.

What changes

  • Unsigned uploads → Signed via Edge Fn
  • Client-side phone reveal → RLS gated
  • AppContext → React Query + Zustand
  • Bare meta tags → react-helmet-async SEO

What stays

  • Vite + React SPA (no framework migration)
  • All existing components reused
  • Supabase tables + RLS policies
  • Tailwind + shadcn/ui
  • Leaflet maps, Framer Motion
  • Cloudinary as image CDN

What gets added

  • Inquiries + messaging system
  • Email notifications (Resend)
  • Razorpay payment integration
  • Listing expiry cron + renewal

Critical RLS Policy — Phone Number Protection

Add this policy to properties: owner_phone is excluded from the default SELECT * public policy. Create a separate Supabase RPC function reveal_owner_phone(property_id) that (1) checks the caller is authenticated, (2) inserts a row into contact_reveals, (3) checks monthly reveal quota, (4) returns the phone number. This single change closes your biggest security gap.

06 — Google Sign-In + Onboarding

First Impression Counts

The current auth flow drops users directly to the dashboard after Google sign-in with no context. Production onboarding answers two questions: who are you? and what do you want to do? This drives better SEO (role data), better personalisation, and measurably higher retention.

Auth Implementation

  • Keep Google OAuth via Supabase Auth (no changes needed)
  • After first sign-in, check profiles.onboarding_complete
  • If false → redirect to /welcome onboarding flow
  • Store role in profiles table: 'tenant' | 'owner' | 'both'
  • Phone number collected in onboarding (not Google profile)

Supabase Trigger — Auto-create Profile

Add a PostgreSQL trigger: AFTER INSERT ON auth.users → automatically inserts a row into profiles with default values. This means you never have a user without a profile record, eliminating an entire class of null-pointer bugs in your frontend.

The One Metric That Drives Everything

Focus on listings per city block / neighbourhood. Density creates the network effect. A tenant searching "2BHK in Koramangala" who finds 15 options on TownHall vs 2 on a competitor becomes a loyal user. Launch in one neighbourhood, make it the most comprehensive rental source for that area, then expand outward. This is exactly how Zomato won — restaurant by restaurant, neighbourhood by neighbourhood.

08 — 30-Day Sprint to Production

Ship in Four Weeks

Minimum work, maximum production-readiness. Every task is doable solo. Weeks are ordered by risk reduction, not feature value.

W1
Security
Fix phone number exposure — Remove owner_phone from public SELECT policy. Create reveal_owner_phone() RPC with auth check + contact_reveals insert + quota check.
Signed Cloudinary uploads — Create Supabase Edge Function sign-upload that generates a signed upload URL. Remove VITE_CLOUDINARY_UPLOAD_PRESET from client bundle.
Add profiles table — Create table + trigger on auth.users insert. Map onboarding_complete, role, reveal_credits columns.
Add listing_reports table + UI — Simple "Report this listing" button on property detail. Feeds admin review queue.
Rate-limit view_count increments — Use Upstash Redis to rate-limit to 1 increment per IP per listing per 24h. Prevents gaming the ranking.
W2
Communication
Inquiry system — Create inquiries table. Add inquiry form modal on property detail. Edge Function sends email to owner via Resend on new inquiry.
Listing expiry cron — pg_cron job runs nightly: SET status='expired' WHERE expires_at < now() AND status='active'. Trigger Resend email to owner with renewal CTA.
7-day expiry warning — Second cron: send warning email to owners whose listing expires in 7 days. Include upgrade CTA to Premium.
Saved properties — saved_properties table + heart button on PropertyCard. Tenant's /saved page listing all wishlisted properties.
Onboarding flow — /welcome page: 3-step role + city + preferences. Sets profiles.onboarding_complete = true. ProtectedRoute redirects first-time users here.
W3
SEO & Perf
Add react-helmet-async for per-page SEO — Install and wrap the app in HelmetProvider. Add dynamic <title>, <meta og:title>, <meta og:image>, <meta og:description> to PropertyDetailPage using Supabase-fetched data. No framework migration required.
Sitemap.xml + robots.txt — Vercel serverless function that queries Supabase for all active property IDs and outputs a valid sitemap. Add static robots.txt pointing to it. Submit both to Google Search Console on launch day.
JSON-LD structured data — Add RealEstateListing schema.org markup to each property page via react-helmet-async. Enables rich snippets in Google Search without any SSR requirement.
Replace AppContext with TanStack Query — Remove the "fetch everything on mount" pattern. Implement paginated queries with stale-while-revalidate. Critical for performance at 1K+ listings.
Add Sentry + Vercel Analytics — 30-minute setup each. You need real error data before launch, not after.
W4
Revenue
Razorpay integration — Replace mock PaymentPage with live Razorpay Checkout (API keys ready). Edge Function verifies payment signature server-side before setting plan_type and extending expires_at.
Plan selection UI — Replace the current single payment button with a 3-column Free / Featured / Premium plan selector. Show ranking boost benefit clearly.
Contact reveal credits — Wire reveal_credits to UI. Show "2 reveals remaining" on detail page. "Get more" CTA links to Razorpay credits purchase.
Owner Dashboard analytics — Add view_count graph, inquiry count, days remaining badge, and "Upgrade Plan" CTA to the dashboard. Owners need to see ROI.
Launch checklist — Custom domain on Vercel, Google Search Console setup, test full owner + tenant flow end-to-end in production, confirm Razorpay webhooks firing correctly.

The Architect's Final Word

You have built a better foundation than most funded startups have at their seed round. The React + Supabase + Vercel stack scales to millions of users on free tiers. The ranking algorithm is sound. The schema is clean. What you need now isn't a rewrite — it's trust infrastructure (security fixes), growth infrastructure (SEO meta tags + sitemap), and revenue infrastructure (Razorpay + plans). In that exact order. Ship Week 1 fixes before you tell anyone the product exists. Everything else can iterate. Security cannot.
